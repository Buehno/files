Lógica do processo
A avaliação é dividida em duas partes sequenciais: Parte 1 — Base comum (30 min) e Parte 2 — Desafio prático por trilha (30 min). A Parte 1 inclui 12 questões conceituais e um mini-desafio integrador de debug. Na Parte 2, o candidato escolhe uma das três trilhas. Isso garante que quem domina apenas uma disciplina ainda mostre excelência nela, enquanto quem domina o conjunto completo é diferenciado pela qualidade da solução integrada.

SQL + Git — 4 questões
S1
Na query SELECT fornecedor, SUM(valor) FROM pedidos WHERE SUM(valor) > 10000 GROUP BY fornecedor, qual é o erro lógico?
A GROUP BY deve vir antes de WHERE na sintaxe SQL
B Funções de agregação não podem ser usadas em WHERE — deve-se usar HAVING
C SUM(valor) não é compatível com GROUP BY fornecedor
D Falta ORDER BY para a query funcionar corretamente
S2
Qual é a diferença principal entre INNER JOIN e LEFT JOIN?
A INNER JOIN é otimizado para grandes volumes; LEFT JOIN para tabelas pequenas
B LEFT JOIN retorna todos os registros da tabela esquerda, com NULL onde não há correspondência
C INNER JOIN trata colunas NULL; LEFT JOIN as ignora automaticamente
D São equivalentes, apenas com sintaxes alternativas

A Cria uma cópia da main com o nome feature-rebase
B Reverte todos os commits da feature ao estado atual da main
C Recria os commits da feature sobre o topo atual da main, linearizando o histórico
D Mescla feature na main criando um commit de merge
S4
Qual das opções segue corretamente o padrão Conventional Commits?
A update: code changes in queries module
B COMMIT — alteração no módulo de pedidos v2
C feat: adiciona filtro de fornecedores por categoria de risco
D fixed bug in supplier queries
